Best overall app build kit
Create a complete downloadable local app for: [APP IDEA].

Use the best development approach for the app idea. Do not force batch files unless they are useful as optional launchers. Do not give copy-and-paste code only; provide the actual files needed to run it.

Before creating files, define:
- Recommended technology choice and why
- File structure
- App state model
- Performance risks
- Edge cases
- Acceptance criteria

Build requirements:
- Polished modern GUI
- Resizable window
- Normal taskbar behavior
- Custom title bar only if it does not break normal Windows behavior
- Drawn minimize/maximize/close icons, not letters or weird symbols
- Clear loading/progress states
- Long tasks run in background workers
- Friendly errors for users
- Detailed logs for debugging
- Hidden developer/debug menu with useful tools
- Settings saved outside the code
- README and debug README
- Test checklist

After building, tell me exactly which file to launch and what to test first.
Massive media folder app kit
Create a local photo/video management app that can handle a massive folder of media.

Core requirements:
- Read common photo and video formats
- Sort by date created/taken when available, with fallback rules
- Apple Photos-style grid with thumbnails
- Fast scrolling using virtualized loading
- Background folder scan
- Progress bar with file counts
- Cancel scan option
- Thumbnail disk cache
- Prioritize visible thumbnails first
- Video preview thumbnails when possible
- Click media to open a clean preview pop-up
- Timeline by date, not just time
- Search and filters
- Friendly skipped-file list
- Debug log export

Performance requirements:
- Do not freeze or show Not Responding during scanning
- Do not load all media into memory
- Do not generate all thumbnails on the UI thread
- Show partial results while loading

Provide the files, README, debug launcher, and test checklist.
Drive mapping/internal IT tool kit
Create a local internal IT tool for managing mapped drives.

Requirements:
- Detect existing mapped drives
- Add/remove drives
- Change drive letters safely
- Prevent duplicate drive letters
- If a conflict exists, prompt to resolve it
- Search shared folders when adding drives
- Validate paths before saving
- Detect duplicate paths and ask before removing duplicates
- Open the linked batch/script when double-clicking a mapped folder path
- Keep available drive letters updated immediately
- Logs for every change
- Backup before modifying files
- Undo/rollback where practical
- Friendly errors for disconnected VPN, missing permissions, unavailable servers, and locked files
- Clean, resizable GUI
- Hidden debug menu

Do not store passwords in plain text. Use the safest practical approach for credentials.
Fix my existing app kit
Fix my existing app without rebuilding it from scratch.

Problem:
[DESCRIBE PROBLEM]

Expected behavior:
[DESCRIBE EXPECTED BEHAVIOR]

Current files/notes:
[PASTE OR ATTACH DETAILS]

Rules:
- Preserve working features
- Make the smallest safe change
- Add logs near the broken behavior
- Improve user-facing error messages
- Do not introduce command windows unless debug launcher is used
- Do not change the visual style unless required
- Explain changed files
- Include a test checklist
- Be honest about anything that could not be fixed

Ask ChatGPT to challenge your prompt
Before answering my request, challenge my prompt.

Request:
[PASTE REQUEST]

Tell me:
- What is unclear
- What might cause bad output
- What constraints are missing
- What I may be over-specifying
- What I may be under-specifying
- What assumptions you will make if I do not clarify

Then rewrite my request into a stronger prompt and proceed using the improved version.
Senior reviewer mode
Act as a senior product designer, senior Windows app developer, and practical IT support lead.

Review this idea:
[PASTE IDEA]

Give me:
- What is useful
- What is risky
- What is too complicated
- What should be simplified
- What normal users will misunderstand
- What needs logs/progress/errors
- What should be configurable
- What should be built first
- What should wait
- A stronger build prompt I can paste back to you
Use these as strong constraints, checklists, and acceptance criteria. No prompt can override safety rules, tool limits, or unavailable information, but these are designed to remove ambiguity and make the output much more likely to match what you intended.
