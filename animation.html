1. Motion basics

Core animation types used everywhere: buttons, cards, modals, menus, images, dashboards, landing pages, and app interfaces.

Fade

Softly changes opacity. Best for clean, calm, professional UI where motion should not distract.

Prompt: Use subtle fade-in and fade-out animations with smooth easing and no harsh movement.

Slide

Moves elements from one side or direction. Great for drawers, cards, menus, alerts, and onboarding screens.

Prompt: Add smooth slide-in animations from the bottom/right with a polished app-like feel.

Scale / Zoom

Grows or shrinks an element. Useful for modals, selected cards, image previews, and hover effects.

Prompt: Use gentle scale animations for cards and popups, with a smooth zoom-in effect on open.

Rotate

Rotates icons, loaders, toggles, badges, and decorative elements. Use lightly for professional designs.

Prompt: Add small rotation animations to icons and decorative UI pieces without making it feel playful.

Bounce

Playful movement that lands with energy. Best for fun apps, success moments, empty states, and badges.

Prompt: Use a light bounce animation for important success states, but keep it refined and not childish.

Spring

Feels natural and physical, like iOS motion. Excellent for modern mobile-style interfaces.

Prompt: Use spring-based motion similar to Apple/iOS, with natural easing and soft overshoot.

Flip

Turns a card or panel around. Good for flash cards, preview/details cards, settings tiles, and galleries.

Prompt: Add a smooth 3D card flip animation for switching between summary and details.

Blur reveal

Starts blurry then sharpens. Good for premium hero sections, glass UI, images, and dramatic entrances.

Prompt: Use a premium blur-to-sharp reveal with opacity fade and slight upward movement.

Glow pulse

Creates a soft light pulse. Great for active states, AI features, audio apps, scan buttons, and focus highlights.

Prompt: Add a soft glowing pulse around the primary action, subtle and modern.

2. Reveals and entrances

These make pages feel polished when content enters, scrolls into view, or loads in a sequence.

Clip reveal

Content appears as if a mask slides open. Great for section headings, image cards, and modern portfolios.

Prompt: Reveal major headings and image panels with a clean masked clip animation.

SVG draw

Lines draw themselves on screen. Great for icons, process flows, maps, signatures, and custom illustrations.

Prompt: Use animated SVG line drawing for icons and process steps, with clean stroke animation.
Hello motion

Typewriter

Text types in character by character. Best for developer tools, terminal effects, AI assistants, and intros.

Prompt: Add a tasteful typewriter text effect for the headline or command-style messages.

Scroll reveal

Elements animate as the user scrolls. Common for landing pages, dashboards, and marketing pages.

Prompt: Animate sections as they scroll into view with fade, slight lift, and staggered card timing.

Modal entrance

Popup opens with opacity, scale, and movement. Good for previews, dialogs, upload panels, and alerts.

Prompt: Make modals open with a soft scale-up, fade-in, and blurred overlay behind them.
Saved successfully

Toast / notification

Small message slides in, stays briefly, then leaves. Perfect for save confirmations and background tasks.

Prompt: Use clean toast notifications that slide up, fade in, pause, then fade out.

3. Micro-interactions

Small details that make an app feel alive: hover states, focus states, button feedback, toggles, and icons.

Hover lift

Cards gently rise when hovered. A must-have for modern dashboards and clean web apps.

Prompt: Give cards a subtle hover lift with soft shadow, no aggressive movement.

Magnetic hover

Button or card slightly follows the cursor. Great for high-end portfolio or landing page interactions.

Prompt: Add magnetic hover behavior to primary buttons so they feel premium and interactive.

3D tilt

Card tilts in perspective. Useful for product cards, gallery thumbnails, and game-like interfaces.

Prompt: Add a subtle 3D tilt hover effect to cards, with perspective and smooth reset.

Elastic press

Button squishes when pressed. Adds physical feedback without needing a heavy animation library.

Prompt: Make buttons slightly compress on click, then spring back naturally.

Error shake

Small shake used for validation errors. Should be quick and limited to the affected field.

Prompt: Use a small horizontal shake only for invalid fields or failed actions.
99%

Counter animation

Numbers count upward or pulse. Excellent for stats, progress, dashboard metrics, and summaries.

Prompt: Animate dashboard numbers counting up smoothly when the section appears.

4. Loading and progress

Use these when importing, scanning, processing, uploading, downloading, searching, or waiting on data.

Spinner

Classic loading indicator. Use for quick operations only. Pair with text for clarity.

Prompt: Add a clean spinner with a friendly loading message for short tasks.

Skeleton shimmer

Placeholder blocks shimmer while content loads. Best for image grids, cards, tables, and feeds.

Prompt: Use skeleton loading cards with a soft shimmer effect while media loads.

Dot wave

Three-dot animated waiting state. Good for chat, AI thinking, search, and compact loading areas.

Prompt: Add a small three-dot wave loading indicator for background thinking or searching.

Progress pulse

A line pulses while something is happening. Good when exact progress is unknown.

Prompt: Use an indeterminate progress bar when exact progress cannot be measured.

Progress fill

A bar fills from left to right. Use when the app can measure scanning, importing, or exporting.

Prompt: Add a real progress bar with percentage, status text, and completed step labels.
Processing folder...

Status message

Plain-language status updates. Very important for local apps handling large folders or slow files.

Prompt: Show friendly status messages for scanning, importing, saving, and errors.

5. Page, layout, and navigation motion

These help whole screens feel connected instead of suddenly changing.

Page transition

A whole screen slides/fades when changing routes. Great for multi-page websites and apps.

Prompt: Use smooth page transitions with fade and slight slide between major screens.

Parallax layers

Different layers move at different speeds. Great for hero sections and immersive landing pages.

Prompt: Add subtle parallax depth to the hero section using layered shapes and images.

Layout morph

One shape or layout smoothly transforms into another. Useful for expanding cards and detail views.

Prompt: Make selected cards smoothly morph into a larger detail panel.

Drawer / sidebar

Navigation panel slides from the side. Should feel fast, stable, and easy to close.

Prompt: Add a side drawer that slides in smoothly with a dimmed blurred background overlay.

Accordion expand

Sections open by expanding height. Useful for FAQs, settings panels, and grouped controls.

Prompt: Use smooth accordion expand/collapse motion with no content jumping.

Sticky scroll effects

Elements pin, shrink, fade, or transform while scrolling. Great for stories and product pages.

Prompt: Add sticky scroll-based section transitions that feel smooth and lightweight.

6. Visual and decorative effects

These are more stylized. Use them to define a strong visual identity, but keep accessibility and performance in mind.

Animated gradient

Gradient colors slowly move. Great for modern SaaS, AI, music, and creative tools.

Prompt: Use a slow animated gradient background with a premium soft glow style.

Moving background

Background blobs, lights, or patterns drift gently. Creates depth without moving the main content.

Prompt: Add very subtle animated background blobs that move slowly behind the interface.

Particles

Small floating dots, sparks, or pixels. Good for audio, gaming, AI, upload success, and celebrations.

Prompt: Add lightweight particles for atmosphere, but keep them minimal and performance-friendly.

Liquid / blob

Organic shape morphing. Good for creative websites, music apps, and friendly modern designs.

Prompt: Use soft liquid blob animations in the background for a friendly modern feel.

Neon / cyber glow

Bright glowing edges and light trails. Best for gaming, music, nightlife, and cyberpunk designs.

Prompt: Create a dark neon cyber style with glowing edges, animated light pulses, and glass panels.

Glassmorphism motion

Blurred panels, soft borders, and moving light. Works well for Apple-like clean app interfaces.

Prompt: Use glassmorphism panels with subtle floating motion, soft shadows, and blurred background light.

Master prompt to ask for a specific animation style

Copy this and replace the bracketed parts with what you want.

Create a modern web interface using [animation style]. The animation should feel [premium / Apple-like / playful / futuristic / minimal / cinematic / enterprise]. Use [fade / slide / scale / spring / blur / parallax / shimmer / hover lift / page transitions] where appropriate. Keep the motion smooth, lightweight, and polished. Use natural easing, staggered timing for repeated cards, and avoid anything too distracting. Include loading states, hover states, active states, focus states, and reduced-motion support. Make the design responsive and make sure nothing overlaps, trails off-screen, or feels cluttered.

Animation rules to include in prompts

These details help keep the result polished, user-friendly, and not annoying.

RuleWhat to ask forWhy it matters
Use natural easingUse smooth ease-out or spring easing instead of linear movement.Linear motion feels robotic and cheap.
Keep motion shortMost UI animations should be around 150–400ms.Long animations make apps feel slow.
Stagger repeated itemsCards should enter one after another with a tiny delay.Lists and grids feel more polished.
Show loading feedbackUse progress bars, skeletons, status text, or spinners for slow work.Users should never wonder if the app froze.
Support reduced motionAdd prefers-reduced-motion support.Important for accessibility and comfort.
Do not overanimateOnly animate actions, state changes, loading, and important visual moments.Too much motion feels distracting.
Use GPU-friendly propertiesPrefer transform and opacity over animating width, height, top, or left.Keeps the interface smooth and lightweight.
Make states obviousInclude hover, active, focus, disabled, selected, loading, success, and error states.The app feels complete and easier to use.
Simple formula: ask for the visual style, then ask for the motion behavior, then ask for usability rules. Example: “Create an Apple-like media gallery with soft fade-in thumbnails, quick hover zoom, skeleton loading, smooth modal previews, reduced-motion support, and lightweight CSS animations.”
